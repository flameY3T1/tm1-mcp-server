# v12 Jobs / Activity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add v12 (Planning Analytics Engine) Jobs/Activity monitoring — `tm1_list_jobs` + `tm1_cancel_job` — and version-gate the monitoring tool surface so v11 connections show thread tools and v12 connections show job tools.

**Architecture:** A new `MonitoringService.getJobs()/cancelJob()` pair hits the v12 `/Jobs` endpoint (rerooted automatically by the existing v12 connection profile). Two new tools in `get-jobs.ts` register only on v12; the existing thread tools register only on v11. Registration is gated by a new `TM1Client.version` accessor. The build's coverage/README gates are static source scanners, so both tool sets stay in the annotation/output-schema maps and README while only one set registers at runtime.

**Tech Stack:** TypeScript (strict), MCP SDK (`server.tool`), Zod output schemas, vitest.

## Global Constraints

- New TM1 REST calls live in a service under `src/tm1-client/services/` — never a flat client (gate `lint:no-flat-api`).
- Every tool declares readOnly/destructive hints in `src/tools/annotation-map.ts` (gate `lint:annotations`).
- New handler field → add it to the matching strict schema in `src/tools/schemas/` (SDK rejects unknown fields; `additionalProperties:false`).
- Secrets masked via `src/lib/mask-secrets.ts`; never log raw creds. (No new secret surface here.)
- Conventional Commits; one logical change per commit. No real customer/server names in tests or docs.
- `config.version` is `11 | 12` (set by `loadConfig`); fixed for a connection's lifetime.
- Verify with `npm run verify` (typecheck strict + lint gates + tests). CI runs the same.
- Job `ID` is a **string** (`Edm.String`); Thread `ID` is numeric. Cancel action: `POST /api/v1/Jobs('{id}')/tm1.Cancel`.
- After adding tools: `npm run tools:update-readme`.

```ts
  /**
   * List active jobs (v12 "Activity"). The v12 successor to Threads.
   * GET /api/v1/Jobs
   */
  async getJobs(): Promise<Job[]> {
    const response = await this.http.request<{
      value: Array<{
        ID: string;
        Description: string;
        State: string;
        ElapsedTime?: string;
        WaitTime?: string;
        Session?: { ID: string | number; Context?: string; User?: { Name?: string } };
        WaitingOn?: Array<{ ID: string; Description: string; State: string }>;
      }>;
    }>(
      "GET",
      "/api/v1/Jobs?$select=ID,Description,State,ElapsedTime,WaitTime&$expand=Session($select=ID,Context;$expand=User($select=Name)),WaitingOn($select=ID,Description,State)",
    );
    return response.value.map((j) => ({
      id: j.ID,
      description: j.Description,
      state: j.State,
      ...(j.ElapsedTime !== undefined ? { elapsedTime: j.ElapsedTime } : {}),
      ...(j.WaitTime !== undefined ? { waitTime: j.WaitTime } : {}),
      ...(j.Session
        ? {
            session: {
              id: String(j.Session.ID),
              ...(j.Session.Context !== undefined ? { context: j.Session.Context } : {}),
              ...(j.Session.User?.Name !== undefined ? { user: j.Session.User.Name } : {}),
            },
          }
        : {}),
      ...(j.WaitingOn
        ? { waitingOn: j.WaitingOn.map((w) => ({ id: w.ID, description: w.Description, state: w.State })) }
        : {}),
    }));
  }

  /**
   * Cancel a running v12 job.
   * POST /api/v1/Jobs('{id}')/tm1.Cancel
   */
  async cancelJob(jobId: string): Promise<void> {
    await this.http.request<void>("POST", `/api/v1/Jobs('${encKey(jobId)}')/tm1.Cancel`, {});
  }
```

- [ ] **Step 5: Create the job tools `src/tools/operations/get-jobs.ts`**

```ts
import type { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { z } from "zod";
import type { TM1Client } from "../../tm1-client.js";
import { PAGINATION_SCHEMA, paginate } from "../pagination.js";
import { FORMAT_SCHEMA, pageResponse, type Column } from "../format.js";

// v12-only: Jobs replace v11 Threads as the "what is running" surface. On a v11
// connection these tools are not registered (thread tools are instead).
export function registerGetJobs(server: McpServer, tm1Client: TM1Client): void {
  if (tm1Client.version !== 12) return;

  server.tool(
    "tm1_list_jobs",
    "List active jobs (Activity) on a TM1 v12 database — the running tasks that replaced v11 threads. Paginated (default 50/page). (v12 only)",
    { ...PAGINATION_SCHEMA, ...FORMAT_SCHEMA },
    async ({ limit, offset, fetchAll, format }) => {
      const jobs = await tm1Client.monitoring.getJobs();
      const page = paginate(jobs, limit, offset, fetchAll);
      type Row = (typeof jobs)[number];
      const columns: Column<Row>[] = [
        { header: "id", get: (j) => j.id },
        { header: "description", get: (j) => j.description },
        { header: "state", get: (j) => j.state },
        { header: "elapsedTime", get: (j) => j.elapsedTime ?? "" },
      ];
      return pageResponse(page, format, { title: "Jobs", columns });
    },
  );

  server.tool(
    "tm1_cancel_job",
    [
      "Cancel a running TM1 v12 job by its ID. Use tm1_list_jobs to find the ID.",
      "Non-idempotent: cancelling a finished job errors. Before: tm1_list_jobs to confirm the job is still running.",
      "(v12 only)",
    ].join(" "),
    {
      jobId: z.string().describe("Job ID to cancel"),
    },
    async ({ jobId }) => {
      await tm1Client.monitoring.cancelJob(jobId);
      return { content: [{ type: "text" as const, text: JSON.stringify({ success: true, jobId }) }] };
    },
  );
}
```

// The client-abort recovery hint. v12 surfaces running work as jobs; v11 as
// threads — point the caller at the tools that exist on their connection.
export function abortHint(version: 11 | 12): string {
  const monitor =
    version === 12
      ? "Use tm1_list_jobs to check for it and tm1_cancel_job to stop it."
      : "Use tm1_list_threads to check for it and tm1_cancel_thread to stop it.";
  return `Request aborted by the client — the process was NOT confirmed failed and may still be executing. ${monitor} Do NOT blindly re-run: that risks a duplicate execution.`;
